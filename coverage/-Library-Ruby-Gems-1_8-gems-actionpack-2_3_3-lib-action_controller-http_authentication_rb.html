246       # Uses an MD5 digest based on time to generate a value to be used only once.
 
247       #
 
248       # A server-specified data string which should be uniquely generated each time a 401 response is made.
 
249       # It is recommended that this string be base64 or hexadecimal data.
 
250       # Specifically, since the string is passed in the header lines as a quoted string, the double-quote character is not allowed.
 
251       #
 
252       # The contents of the nonce are implementation dependent.
 
253       # The quality of the implementation depends on a good choice.
 
254       # A nonce might, for example, be constructed as the base 64 encoding of
 
255       #
 
256       # => time-stamp H(time-stamp ":" ETag ":" private-key)
 
257       #
 
258       # where time-stamp is a server-generated time or other non-repeating value,
 
259       # ETag is the value of the HTTP ETag header associated with the requested entity,
 
260       # and private-key is data known only to the server.
 
261       # With a nonce of this form a server would recalculate the hash portion after receiving the client authentication header and
 
262       # reject the request if it did not match the nonce from that header or
 
263       # if the time-stamp value is not recent enough. In this way the server can limit the time of the nonce's validity.
 
264       # The inclusion of the ETag prevents a replay request for an updated version of the resource.
 
265       # (Note: including the IP address of the client in the nonce would appear to offer the server the ability
 
266       # to limit the reuse of the nonce to the same client that originally got it.
 
267       # However, that would break proxy farms, where requests from a single user often go through different proxies in the farm.
 
268       # Also, IP address spoofing is not that hard.)
 
269       #
 
270       # An implementation might choose not to accept a previously used nonce or a previously used digest, in order to
 
271       # protect against a replay attack. Or, an implementation might choose to use one-time nonces or digests for
 
272       # POST or PUT requests and a time-stamp for GET requests. For more details on the issues involved see Section 4
 
273       # of this document.
 
274       #
 
275       # The nonce is opaque to the client. Composed of Time, and hash of Time with secret
 
276       # key from the Rails session secret generated upon creation of project. Ensures
 
277       # the time cannot be modifed by client.
 
278       def nonce(time = Time.now)
 
279         t = time.to_i
 
280         hashed = [t, secret_key]
 
281         digest = ::Digest::MD5.hexdigest(hashed.join(":"))
 
282         Base64.encode64("#{t}:#{digest}").gsub("\n", '')
 
283       end
